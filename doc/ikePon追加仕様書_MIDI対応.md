# ikePon 追加仕様書：MIDIコントロール機能の導入（ドラムパッド最適化版）

本文書は、アプリケーション全体（パッド、グローバルアクション、バンク、ミキサー）を外部MIDI機器（特にドラムパッド・グリッドコントローラー）から遠隔制御するための「MIDIコントロール機能」に関する追加仕様についてまとめたものである。

---

## 1. システム設定への拡張（UI）

### 1.1 MIDIデバイス選択欄の追加

* **配置位置**: 設定ダイアログ（`Ctrl + E`）内に「MIDI入力デバイス」のプルダウンメニュー（ComboBox）を新設する。ウィンドウ内の場所は、「PAセパレートモード」と「インターロック」の間。
* **挙動**:
* 起動時にシステムに接続されているMIDI入力デバイスを自動検出し、リストに列挙する。
* デバイスが選択された場合、該当デバイスからのMIDI信号の常時監視（リスニング）を開始する。
* 「なし（無効）」の選択肢も用意し、初期値とする。
* アプリ実行中にMIDIデバイスが抜差しされた場合の切断・再接続ハンドリング（例外防止）を実装する。



---

## 2. MIDI受信基本プロトコル

* **対応チャンネル**: 全チャンネルのイベントを包括して受信する「オムニオン（Omni On）」状態とする。送信側のMIDIチャンネル設定に依存せず、すべてのメッセージを同一に処理する。
* **ベロシティ処理**: ノートオン（Note On）時のベロシティ（Velocity）値は `1 〜 127` を正常に受信する。ただし、値の大小による挙動の変更（音量追従など）は行わず、「値が1以上であればトリガー発火」の単一判定とする。
* **ノートオフの扱い**: `Note Off` イベント、および `Velocity = 0` の `Note On` イベントは、本機能のトリガーとしては原則無視する。

---

## 3. MIDIノートマップ（Note Mapping）

すべてのコントロールを、市販のパッドコントローラー（4×4や2×4のグリッド）と親和性の高い「16刻み（1オクターブ強）の連番ブロック」で管理する。

### 3.1 PADエリア（ノート番号: 36 〜 51）【16個ブロック】

ハードウェア of ドラムパッド等に多く見られる「左下を起点に右上に向かって昇順」の配置レイアウトを採用し、本体の1バンク（16パッド）と完全連動させる。

#### PAD グリッドマップ

```
[48] [49] [50] [51]   ← 1行目：パッド  0 〜  3（1, 2, 3, 4 キー列に対応）
[44] [45] [46] [47]   ← 2行目：パッド  4 〜  7（Q, W, E, R キー列に対応）
[40] [41] [42] [43]   ← 3行目：パッド  8 〜 11（A, S, D, F キー列に対応）
[36] [37] [38] [39]   ← 4行目：パッド 12 〜 15（Z, X, C, V キー列に対応）

```

### 3.2 グローバルアクション ＆ BANKエリア（ノート番号: 52 〜 67）【16個ブロック】

4マス×4列の物理配置に完璧に適合し、直感的なオペレーションが行えるよう、下半分（52〜59）をシステム・画面制御、上半分（60〜67）をBANK切り替え（常用A〜Dを最上段に配置）として1ブロック化する。

#### グローバルアクション ＆ BANK グリッドマップ

```
[64:BANK A  ] [65:BANK B  ] [66:BANK C  ] [67:BANK D  ]  ← 4行目（最上段）：BANK（A〜D）
[60:BANK E  ] [61:BANK F  ] [62:BANK G  ] [63:BANK H  ]  ← 3行目：BANK（E〜H）
[56:FULL SCR] [57:DISPLAY ] [58:（予備） ] [59:（予備） ]  ← 2行目：画面制御
[52:ALL CUT ] [53:ALL FADE ] [54:PAUSE    ] [55:（予備） ]  ← 1行目（最下段）：再生制御

```

#### 各ボタンの挙動詳細

* **52**: **ALL CUT** （即座に全停止）
* **53**: **ALL FADE** （一括フェードアウト）
* **54**: **PAUSE** （一時停止/再開）
* **56**: **FULL SCR** （全画面トグル）
* **57**: **DISPLAY** （動画ウィンドウ開閉トグル）

### 3.3 ミキサーメモリ・ミュートボタン（ノート番号: 68 〜 83）【16個ブロック】

画面上の縦型フェーダーの並びと完璧にシンクロさせるため、「1つの縦列が1つのチャンネル」となる【縦列連動マッピング】を採用する。

#### ミキサーボタン グリッドマップ（縦列連動）

```
[80:MOVIE  M1] [81:BGM    M1] [82:SE     M1] [83:MASTER M1 ]  ← 4行目（最上段）
[76:MOVIE  M2] [77:BGM    M2] [78:SE     M2] [79:MASTER M2 ]  ← 3行目
[72:MOVIE  M3] [73:BGM    M3] [74:SE     M3] [75:MASTER M3 ]  ← 2行目
[68:MOVIE  MU] [69:BGM    MU] [70:SE     MU] [71:MASTER MU ]  ← 1行目（最下段）
  ↑                    ↑                  ↑                  ↑
【MOVIE列】         【BGM列】          【SE列】          【MASTER列】

```

## 4. コントロールチェンジ（CC / フェーダー制御）

外部MIDIコントローラーの物理フェーダーやノブの動きを、ミキサーエリアの全128段階（0〜127）の線形スケールにダイレクトにバインドする。

* **コントロールチェンジ（CC）の受信値**: `0 〜 127` をそのままミキサーのスケール値に適用する。

### CCアサイン一覧

* **CC 16**: **MOVIE** フェーダー
* **CC 17**: **BGM** フェーダー
* **CC 18**: **SE** フェーダー
* **CC 19**: **MASTER** フェーダー

---

## 5. クラス・ファイル構成への影響

本機能の実装にあたり、内部処理用のマネージャー（コントローラー）を新設し、音声エンジンやUIスレッドと非同期で結合する。

```
src/ikePon/
├── Controller/
│   ├── MidiController.cs         [新規] MIDI入力デバイスの監視、メッセージのパース、各コントローラーへのディスパッチ処理
├── Model/
│   └── AppSettings.cs            [拡張] SelectedMidiDeviceName（settings.jsonへの保存用プロパティ）の追加
└── UI/
    └── Dialogs/
        └── SettingsDialog.xaml   [拡張] MIDIデバイス選択用ComboBoxのUI追加

```
